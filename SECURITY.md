## 2. AI Code Audit & Attribution

AI is great at "vibing," but it's notorious for suggesting deprecated or insecure libraries.

**AI Tool Used:** Claude, GitHub Copilot (Claude Haiku, Raptor Mini, etc.)

**The "Veto":** Claude had a requirements.txt file outlining the various dependencies needed by python for the app. These dependencies were given fixed versions that are secure now. I told it to ensure these dependencies are always at their latest versions when the app is built. It did this using pip-tools which generates a new requirements.txt with the latest versions each time the app is built.

---

## 3. Hardening Measures (The "Big Two")

Detail the two specific security features required for the assignment.

| Feature | Implementation Detail | Threat Mitigated |
|---|---|---|
| **Measure #1: Parameterized Queries** | Parameterized queries are used with "?"s as placeholders when users are querying the CVE database. | SQL Injection |
| **Measure #2: Access Control** | Any PoC endpoint requires a "login_required" decorator and valid session tokens. | Broken Access Control |
